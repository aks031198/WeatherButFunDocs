Support Policy

Last updated: April 16, 2026. This page explains how support works for the current version of Weather But Fun and what to include when reporting issues.

Support Channel

Where to ask for help

Developer: Ayush Kumar Sinha

Email: ayush031198@gmail.com

Support requests and bug reports can be sent by email or filed on the public GitHub issue tracker:

github.com/aks031198/Weather-But-Fun/issues

Because issues are public, do not include personal data, exact addresses, or other sensitive information in your report.

What To Include

Make bug reports easier to fix

  • Your device model and iOS version.
  • The app version or build you are using.
  • Whether the problem happened in the app, the widget, or both.
  • The weather scene you expected and what you saw instead.
  • A screenshot or screen recording if the issue is visual.
  • Whether location access was granted and whether the app had been opened recently.

Quick Checks

Before opening a ticket

  • Open the main app at least once and allow location access if you want live nearby weather.
  • Confirm the device has network connectivity for live weather updates.
  • Remove and re-add the widget if the widget looks stale after a major app update.
  • Check iOS widget and background refresh settings if the widget updates slowly.
  • Restart the app after changing system permission settings.

Current App Notes

Behavior that is expected today

  • The app currently requests location access only while you use the app.
  • The widget uses the latest cached snapshot saved by the app and shared through the app group.
  • The free experience is designed around a slower refresh rhythm, while the premium-style setting uses a tighter refresh rhythm.
  • The current codebase does not include live StoreKit purchase handling yet.

Support Scope

What this support policy covers

Support is provided on a best-effort basis for installation issues, permissions, widget syncing, weather refresh behavior, and visual bugs in the current app. Response times are not guaranteed.

Feature requests are welcome, but there is no guarantee that any specific request will be added to the roadmap.

Billing

Subscriptions and purchases

The current build includes premium-oriented messaging and settings, but the current codebase does not yet process live in-app purchases. If paid subscriptions are introduced in a future release through Apple, billing and refunds will generally be handled through Apple under Apple's storefront policies.
